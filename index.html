HTML Reference Tables

Table Tags

Tag Name Description
<table> Table The wrapper element for all HTML tables.
<thead> Table Head The set of rows defining the column headers in a table.
<tbody> Table Body The set of rows containing actual table data.
<tr> Table Row The table row container. Use to start a new row.
<td> Table Data The table data container. Where you type the content for each cell of the table.
<th> Table Header Defines a cell as a head of a group of table cells. By default this text is bold and centered.
<tfoot> Table Foot The set of rows defining the footer in a table.

Table Attributes

Attribute Name Description
colspan Column Span Specifies the number of columns a table cell should span.
rowspan Row Span Specifies the number of rows a table cell should span.

Form Tags and Attributes

Tag Name Description
<form> Form Creates a form to collect information
<input> Input Creates an Input field. Use a name="" attribute for the information to be sent when the form is submitted and use a type="" attribute to determine how it renders on the page:

type attributes
  • "text" = renders a text box for a user to type into.
  • "password" = will replace the input text with another character like an asterisk (*) or a dot (â€¢).
  • "number" = can restrict what is typed into input field to just numbers and a few special characters. We can also provide a "step" attribute which creates arrows inside the input field to increase or decrease by the value of the step attribute.
  • "range" = creates a slider. Can assing values to the "min" and "max" attributes of the <input>. Can also use the "step" attribute to control how fluid the slider works (smaller value= more fluid).
  • "checkbox" = Each <input> uses the same value for the "name" attribute. This groups the inputs together. Each <input> has a unique "id" to pair with a <label>. There are assigned values to the "value" attribute of the checkboxes. These values are not visible on the form. Use an associated <label> "for" to identify the checkbox.
  • "radio = like "checkbox", but only one can be selected (from those with same "name" attribute).
  • "submit" = creates a submit button.
<label> Label Has an opening and closing tag. Displays text that is written between opening and closing tag. To associate a <label> and <input>, the <input> needs an "id" attribute. We then assign the "for" attribute of the <label> element with the value of the "id" attribute of <input>.
<select> & <option> Datalist Input select needs "id" and "name" attributes ("id" to link to "for" attribute of <label>). The <option> elements go between the <select> tags. The text that is rendered in the list is the text included between the opening and closing <option> tags. However, it is the value of the "value" attribute that is used in form submission (and "name" attribute of <select>).
<datalist> Creates a dropdown list used with a <input type="text" list="LIST NAME"> element ("id" attribute of <datalist> is associated with "list"). The input creates a text field that users can type into and filter options from the datalist. Then use <option> elements like dropdown lists (don't need text in between <option> opening and closing tags).
<textarea> Text Area Field used to create a bigger text field. Use "rows" and "cols" attributes to determine size. Any text in between the opening and closing tags will be the default value to the text area.
